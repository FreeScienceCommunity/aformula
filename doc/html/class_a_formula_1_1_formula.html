Detailed Description

A mathematical formula which can be evaluated.

This class, with a relatively simple public API, is the main point of interaction with AFormula. It encapsulates a single mathematical formula, with a given expression and set of variables.

Available formula-execution backends.

Note:
Some of these backends may be surrounded by #if 0 preprocessor blocks on your system. If so, do not remove them! These backends were disabled at compile-time on your system, and support for them is not included in AFormula. Enabling them will cause AFormula-based programs to crash.

If you want to enable more backend support, install LLVM and/or libjit, and consult the --with-llvm and --with-libjit options to the configure script.

Set a variable to a given pointer.

Parameters:
[in] variable Name of the variable to be set.
[in] pointer Pointer to the double value for this variable.
Returns:
True if the variable could be properly registered, false otherwise.

This function registers a new variable for use in a formula. Since variables in a formula are read/write (that is, we have an assignment operator), you must supply a pointer to a double value which holds the current value of the given variable, and will hold any changes to the variable which are caused by evaluating the formula.

Note:
The pointer passed in here must be a pointer to a static (or global) double! The various JIT compilers are not able to access the stack space of other functions. To repeat: you must declare any double passed in as a pointer here as static.

A variable may fail to be registered if the variable has the same name as another variable, one of the AFormula built-in functions, or one of the built-in mathematical constants.
